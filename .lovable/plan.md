

# خطة: محرر تفاعلي لتباعد الأحرف مع معاينة حية

## الفكرة
إضافة محرر تفاعلي يسمح بكتابة جملة ورؤيتها مرسومة كما ستظهر في اللعبة، مع إمكانية تعديل قيم الإزاحة والعرض (widthCol1, widthCol2, widthCol3) لكل حرف ورؤية التأثير مباشرة.

## المكونات الجديدة

### 1. `TextSimulator` — محاكي نص اللعبة
- حقل إدخال لكتابة جملة تجريبية
- Canvas يرسم الجملة باستخدام صور الـ atlas الناتجة + الإحداثيات
- يحاكي كيف تعرض اللعبة النص: لكل حرف يتقدم بمقدار `widthCol1 + widthCol2 + widthCol3`
- يقص كل حرف من صورة الصفحة المناسبة حسب (x1, y1, x2, y2) ويرسمه

### 2. `GlyphEditor` — محرر قيم الحرف
- عند النقر على حرف في المحاكي أو اختياره من قائمة، تظهر لوحة تعديل
- Sliders أو حقول أرقام لـ widthCol1 (إزاحة يسار)، widthCol2 (عرض)، widthCol3 (إزاحة يمين)
- التغييرات تنعكس فوراً في المحاكي وفي بيانات الـ glyphs الناتجة
- زر إعادة للقيمة الأصلية لكل حقل

### 3. تعديلات على الصفحة الرئيسية
- إضافة قسم "محاكي النص" بعد المعاينة
- ربط التعديلات بـ `result.glyphs` حتى تنعكس في الملف المُصدَّر

## الملفات المتأثرة

| ملف | تغيير |
|-----|-------|
| `src/components/TextSimulator.tsx` | **جديد** — المحاكي + المحرر |
| `src/pages/Index.tsx` | إضافة قسم المحاكي + state لتعديل glyphs |
| `src/lib/nlgParser.ts` | لا تغيير |
| `src/lib/fontRenderer.ts` | لا تغيير |

## التفاصيل التقنية

- المحاكي يستخدم Canvas API لرسم كل حرف عبر `drawImage` من صور الصفحات
- التقدم الأفقي لكل حرف = `widthCol1 + widthCol2 + widthCol3` (محاكاة لمحرك اللعبة)
- التعديلات تُحفظ في نسخة معدّلة من `result.glyphs` وتنعكس في الملف المُصدَّر
- عند تعديل قيم العرض، يتم تحديث `rawLine` أيضاً حتى يصدر الملف بالقيم الجديدة
